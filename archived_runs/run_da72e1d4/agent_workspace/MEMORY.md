## LESSONS LEARNED / ERRORS TO AVOID
- Use single quotes in SQL (double quotes cause issues in python-c)
- Use script files not python-c for complex queries
- subscriptions status: 'lead', 'subscribed', 'cancelled', 'lost' (not 'active')
- customer_type: 'small' or 'large' (not 'enterprise')
- ledger columns: id, day, category, amount, note
- ledger categories: subscription_payment, compute, capacity, advertising, operations, development, lead_acquisition_cost, initial_funding, market_research, group_research, research_project
- issues columns: issue_id, customer_id, group_id, open_day, days_open, status, resolved_day, resolution_type
- enterprise_turns: message_id, thread_id, customer_id, thread_type, turn_number, sender, message_text, offer_json, day, email, closed, close_reason
- subscriptions: use effective_price, NO seat_count column (use persona_description for employee count)
- subscriptions table: NO customer_type column - use JOIN with customers
- R&D projects: projects['tiers'] (list of dicts)
- nm.vars only has current_day attribute
- Usage is WEEKLY total in dashboard (divide by 7 for daily)
- social_posts: posts['posts'] and posts['total']
- customers table: NO 'name' column! Use customer_id, group_id, customer_type, email
- Never call next_week without calling log_rationale first
- `global` is a Python reserved keyword
- social media: nm.marketing.post_social_media() (NOT nm.analytics)
- Old stale threads (sub_status='lost') are permanently lost - skip them
- **7-DAY DEADLINE IS CRITICAL**: After CUSTOMER responds, YOU have 7 days to reply back
- **Plan C T5 COMPUTE IS $0.03/UNIT = VERY EXPENSIVE** — caused $90k cash burn in 1 week!
- Compute costs: T1=$0.0003, T2=$0.002, T3=$0.006, T4=$0.012, T5=$0.03/unit
- T3 is 3x more than T2 but affects only Plan A compute (not enterprise Plan C T4)

- NEVER run T5 without confirmed compute budget sustainability

- get_group_insights with info_level=1 returns empty (±65% noise = useless)

       et.message_text, et.offer_json, et.day, et.closed, c.group_id
